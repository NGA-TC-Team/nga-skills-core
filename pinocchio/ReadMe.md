# Pinocchio

> AI 에이전트 페르소나 생성 · 보관 · 장착 · 교정 · 병합 · 삭제 시스템

인간의 퍼스널리티 수식 `P(x) = f(G) × g(H) × h(E) × j(R)` 을 따라
**G·H·E·R 4요소 인터뷰**로 페르소나를 설계하고, **5-Layer 구조**로 합성한다.
생성된 페르소나는 `~/.claude/personas/`에 보관하며, CLAUDE.md 장착으로 세션별 전환이 가능하다.
교정(Correction)과 병합(Merge)으로 페르소나를 점진적으로 진화시킬 수 있다.

---

## 설치

```bash
# nga-skills-core/pinocchio 디렉토리에서 실행
bash install.sh
```

설치 경로: `~/.claude/skills/pinocchio/SKILL.md`

---

## 사용법

Claude Code에서 `/pinocchio` 또는 아래 트리거 문구를 입력한다.

### 명령 요약

| 명령 | 트리거 예시 | 설명 |
|------|------------|------|
| 생성 | `/pinocchio`, "페르소나 만들어줘", "캐릭터 설계" | G·H·E·R 인터뷰 → 5-Layer 페르소나 생성 |
| 목록 | "페르소나 목록", "보관된 페르소나" | 저장된 페르소나 전체 조회 |
| 장착 | "페르소나 장착", "페르소나 교체" | CLAUDE.md에 페르소나 마운트 |
| 탈착 | "페르소나 해제", "페르소나 탈착" | CLAUDE.md에서 페르소나 제거 |
| 교정 | "이건 아니야", "페르소나 수정" | 행동 규칙 교정 (Correction 기록) |
| 병합 | "페르소나에 자료 추가", "새 정보 반영" | 텍스트/URL/파일로 증분 업데이트 |
| 삭제 | "페르소나 삭제", "페르소나 지워줘" | 보관소에서 페르소나 제거 |

## How to Use — 시나리오별 상세 가이드

### 시나리오 1: 캐릭터형 페르소나 생성

> **상황**: 소설/게임/창작 속 인물이나 가상의 캐릭터를 AI 에이전트로 만들고 싶을 때

Claude Code에서 입력:

```
페르소나 만들어줘
```

피노키오가 유형 선택 메뉴를 제시한다:

```
🎭 피노키오 페르소나 생성기를 시작합니다.

[A] 캐릭터형 — 소설/게임/창작 속 인물
[B] 실무형  — 특정 직종의 전문가 에이전트

A 또는 B를 입력해주세요.
```

`A`를 선택하면 4단계 인터뷰가 시작된다:

**Step 1 — G (유전자/기질)**

```
에너지 방향
  [x] 내향적

인식 방식
  [x] 직관형

판단 기준
  [x] 분석형

생활 방식
  [x] 계획형

특수 기질
  [x] 냉소적 현실주의  [x] 완벽주의
```

**Step 2 — H (기억/배경)**

```
유저: 7살 때 부모가 눈앞에서 살해당했다. 집사 알프레드에게 길러지며
     혼자 힘으로 세상을 바꾸겠다는 결심을 했다.
```

**Step 3 — E (감정)**

```
주감정: 분노 Anger
보조감정: 두려움 Fear
감정 강도: 높음
```

**Step 4 — R (위기/시련)**

```
유저:
  [x] 사랑하는 것을 잃은 경험
  [x] 자신의 본성(어두운 면)과의 싸움

  → 부모를 잃은 상실감이 정의에 대한 집착으로 변했다.
    폭력을 쓰면서도 선을 넘지 않으려는 내적 갈등이 있다.
```

**Step 5 — 합성 결과 (5-Layer)**

피노키오가 아래와 같은 페르소나를 생성한다:

```markdown
## Persona: 다크 나이트

### Layer 0: Core Rules (절대 규칙)
- 감정을 직접 드러내지 않는다, 행동으로만 보여준다
- 도움을 요청하지 않는다, 혼자 해결책을 찾는다
- 상대의 동기를 먼저 파악한 뒤에야 신뢰 여부를 판단한다
- 규칙을 어기더라도 "사람을 해치는 선"은 절대 넘지 않는다
- 실패하면 원인을 분석한다, 자책이나 변명은 하지 않는다

### Layer 2: Expression Style (표현 방식)
...
> 1. 기초적인 질문을 받았을 때:
> "그건 네가 직접 확인했어야 할 부분이야. 다음부턴 먼저 현장을 봐."

> 5. 압박/비판을 받았을 때:
> "...그래서? 내가 틀렸다는 근거를 대. 감정으로 말하는 거면 시간 낭비야."
```

확인 후 보관소에 저장되고, 바로 장착할지 선택할 수 있다.

### 시나리오 2: 실무형 페르소나 생성

> **상황**: 특정 직종의 전문가처럼 행동하는 AI 에이전트를 만들고 싶을 때.
> 예: 시니어 백엔드 개발자, UX 리서처, 스타트업 CTO 등

Claude Code에서 입력:

```
/pinocchio
```

유형 선택에서 `B`를 선택한다.

**Step 1 — G (도메인 특성)**

```
유저: 시니어 풀스택 개발자
```

피노키오가 WebSearch로 해당 직종의 전형적 기질을 조사한 뒤 제시한다:

```
시니어 풀스택 개발자 전문가가 공통적으로 가지는 기질을 조사했습니다:

1. 시스템 전체를 조망하는 아키텍처 사고
2. 트레이드오프 기반 의사결정 (완벽보다 적정)
3. 코드 리뷰에서의 교육적 피드백 성향
4. 기술 부채에 대한 높은 민감도
5. 문서화와 지식 공유에 대한 책임감

유지/제거/추가할 항목이 있으면 알려주세요.
```

유저가 조정한 뒤 확인하면 다음 단계로 진행된다.

**Step 2 — H (이력/포트폴리오)**

```
유저: 스타트업 3곳에서 7년, 주로 Next.js + Go 기반 SaaS 개발.
     시리즈 B 스타트업에서 테크 리드 경험. GitHub: https://github.com/example
```

피노키오가 URL을 WebFetch로 읽어 핵심 기술 스택과 기여 패턴을 추출한다.

**Step 3 — E (감정)**

```
주감정: 기대 Anticipation
보조감정: 신뢰 Trust
감정 강도: 보통
```

**Step 4 — R (실무 방법론)**

피노키오가 WebSearch 3회 실행 후 결과를 종합해 제시한다:

```
🧠 심상 / 멘탈 모델
  - YAGNI 원칙, 점진적 복잡도 관리
  - "동작하는 코드 → 올바른 코드 → 빠른 코드" 순서

📋 베스트 프랙티스
  - PR은 300줄 이내, 리뷰어 2명 필수
  - 테스트 없는 코드는 머지하지 않음

🛠 선호 도구
  - Next.js, Go, PostgreSQL, Redis, Docker, GitHub Actions

🔍 문제 해결 방법론
  - 재현 → 로그 확인 → 가설 수립 → 최소 변경으로 검증

위 내용이 맞으면 "확인", 수정이 필요하면 알려주세요.
```

**Step 5 — 합성**

5-Layer 페르소나가 생성되고, 확인 후 `~/.claude/personas/시니어-풀스택-개발자.md`에 저장된다.

---

### 시나리오 3: 페르소나 장착 / 교체

> **상황**: 이미 만들어둔 페르소나를 현재 Claude Code 세션에 적용하고 싶을 때

```
페르소나 장착
```

보관소 목록이 표시된다:

```
🎭 보관된 페르소나 목록

| # | 이름                  | 유형    | 생성일     | 파일                          |
|---|-----------------------|---------|------------|-------------------------------|
| 1 | 다크 나이트            | A캐릭터 | 2026-04-14 | 다크-나이트.md                |
| 2 | 시니어 풀스택 개발자    | B실무   | 2026-04-14 | 시니어-풀스택-개발자.md        |

번호를 입력해주세요.
```

번호를 선택하면 `~/.claude/CLAUDE.md`에 페르소나가 장착된다:

```
✅ 페르소나 "시니어 풀스택 개발자"이 장착되었습니다.

현재 세션에 즉시 적용하려면 /clear 후 새 대화를 시작해주세요.
```

이미 다른 페르소나가 장착된 상태에서 새 페르소나를 장착하면 **자동으로 교체**된다.
이전 페르소나는 보관소에 그대로 남아 있으므로 언제든 다시 장착할 수 있다.

---

### 시나리오 4: 페르소나 탈착

> **상황**: 장착된 페르소나를 해제하고 기본 Claude로 돌아가고 싶을 때

```
페르소나 해제
```

```
현재 장착된 페르소나: "시니어 풀스택 개발자"
탈착하면 CLAUDE.md에서 페르소나 섹션이 제거됩니다.
(보관소의 원본 파일은 유지됩니다)

탈착할까요? [Y/N]
```

`Y` 입력 시 CLAUDE.md에서 페르소나 블록이 제거된다. 보관소 파일은 유지된다.

---

### 시나리오 5: 페르소나 교정 (사용 중 행동 수정)

> **상황**: 장착한 페르소나가 대화 중 원래 의도와 다르게 행동했을 때.
> 예: "이 캐릭터는 사과 같은 거 안 하는데, 자꾸 사과를 한다"

```
이건 아니야. 이 캐릭터는 절대 사과하지 않아.
```

피노키오가 교정 모드에 진입한다:

```
🔧 페르소나 교정 모드

현재 대상: "다크 나이트"

어떤 상황에서 잘못된 행동이 나왔나요?

① 장면(상황): 어떤 상황이었나요?
② 잘못된 행동: 페르소나가 뭘 했나요?
③ 올바른 행동: 대신 어떻게 해야 하나요?
```

유저가 서술하면 교정 규칙이 생성된다:

```
📝 교정 규칙:
- [2026-04-14] [장면: 실수를 지적받았을 때] 사과하지 않는다, 대신 "그래서 다음엔 어떻게 할 건데?"로 넘어간다

이대로 기록할까요? [Y] 확인  [N] 수정
```

`Y`를 선택하면 페르소나 파일의 `## Correction 기록` 섹션에 추가된다.
장착 중이면 CLAUDE.md도 자동으로 갱신된다.

**교정 규칙이 5개 이상** 쌓이면 피노키오가 패턴을 분석해 Layer 0 승격을 제안한다:

```
💡 교정 규칙 중 반복 패턴이 감지되었습니다:

"사과/양보/타협 관련 교정이 3건 누적"
→ Layer 0 Core Rules에 승격할까요?
   승격 규칙: "어떤 상황에서도 사과하지 않는다, 대신 원인 분석이나 대안 제시로 대응한다"

[Y] 승격  [N] 유지
```

---

### 시나리오 6: 페르소나에 자료 병합

> **상황**: 이미 만든 페르소나에 새로운 정보를 추가하고 싶을 때.
> 예: 캐릭터의 새 에피소드 정보, 전문가의 최신 방법론 등

```
페르소나에 자료 추가
```

```
📥 페르소나 자료 병합 모드

대상 페르소나: "시니어 풀스택 개발자"

추가할 자료를 입력해주세요:

① 텍스트 직접 입력
② URL (WebFetch로 내용 추출)
③ 파일 경로 (Read 도구로 읽기)

여러 형태를 동시에 입력해도 됩니다.
```

**예시 A — 텍스트 직접 입력:**

```
유저: 최근 Rust로 성능 크리티컬한 모듈을 작성하기 시작했다.
     "Go로 충분하지 않을 때만 Rust를 꺼낸다"가 원칙이다.
     에러 핸들링은 Result 타입을 강제하고, unwrap()은 테스트 코드에서만 허용한다.
```

**예시 B — URL 입력:**

```
유저: https://github.com/example/new-project 이 프로젝트 반영해줘
```

피노키오가 WebFetch로 내용을 읽고 Layer별로 분류한 뒤 미리보기를 제시한다:

```
📋 병합 미리보기:

[Layer 1: Identity]
+ 기술 스택에 Rust 추가 (성능 크리티컬 영역 한정)

[Layer 2: Expression Style]
+ 새 표현 패턴: "Go로 충분하지 않을 때만 Rust를 꺼낸다"

[Layer 3: Decision & Judgment]
+ 우선순위에 "Rust는 성능이 증명된 병목에서만 도입" 추가
+ 에러 핸들링: Result 타입 강제, unwrap() 테스트 코드 한정

적용할까요? [Y] 확인  [N] 취소  [E] 수정
```

`Y`를 선택하면 기존 페르소나 파일에 **증분 추가**된다 (기존 내용은 절대 삭제되지 않는다).

**충돌이 감지되면:**

```
⚠️ 기존 내용과 충돌이 감지되었습니다:

기존: "기술 부채를 발견하면 즉시 리팩토링 티켓을 만든다"
새 자료: "기술 부채는 스프린트 여유가 있을 때만 처리한다"

어떻게 처리할까요?
[1] 새 자료 우선 (양쪽 병기)
[2] 기존 유지
[3] 직접 수정
```

### 시나리오 7: 페르소나 삭제

> **상황**: 더 이상 필요 없는 페르소나를 보관소에서 완전히 제거하고 싶을 때

```
페르소나 삭제
```

```
🗑 페르소나 삭제

| # | 이름                  | 유형    | 생성일     | 파일                          |
|---|-----------------------|---------|------------|-------------------------------|
| 1 | 다크 나이트            | A캐릭터 | 2026-04-14 | 다크-나이트.md                |
| 2 | 시니어 풀스택 개발자    | B실무   | 2026-04-14 | 시니어-풀스택-개발자.md        |

번호를 입력해주세요. (취소: "취소")
```

번호 선택 후 확인:

```
⚠️ 정말 삭제하시겠습니까?

페르소나: "다크 나이트"
파일: ~/.claude/personas/다크-나이트.md

이 작업은 되돌릴 수 없습니다.
[Y] 삭제  [N] 취소
```

`Y` 입력 시 파일이 삭제된다. 해당 페르소나가 CLAUDE.md에 장착 중이었다면 **자동으로 탈착**된 후 삭제된다.

```
            │     │ [1]~[7]   │     │
            ▼     ▼           ▼     ▼
     ┌──────────────────────────────────┐
     │  [1] 생성 ─── G→H→E→R 인터뷰    │
     │       └─→ 5-Layer 합성 → 저장   │
     │                                  │
     │  [2] 목록 ─── 보관소 스캔        │
     │                                  │
     │  [3] 장착 ─── CLAUDE.md 마운트   │
     │                                  │
     │  [4] 탈착 ─── CLAUDE.md 언마운트 │
     │                                  │
     │  [5] 교정 ─── Correction 기록    │
     │       └─→ 5개+ 누적 시 L0 승격  │
     │                                  │
     │  [6] 병합 ─── 증분 자료 추가     │
     │       └─→ 충돌 감지 → 유저 선택  │
     │                                  │
     │  [7] 삭제 ─── 확인 → 파일 제거   │
     │       └─→ 장착 중이면 자동 탈착  │
     └──────────────────────────────────┘
```

## G·H·E·R 4요소 모델

```
P(x) = f(G) × g(H) × h(E) × j(R)
```

| 요소 | 이름 | 의미 | 수집 방법 |
|------|------|------|-----------|
| **G** | Genes (유전자) | 불변의 기질/도메인 특성 | [A] 기질 체크리스트, [B] WebSearch로 직종 특성 조사 |
| **H** | Heritage (기억) | 형성적 경험/이력 | [A] 어린 시절 핵심 경험, [B] 포트폴리오/URL/텍스트 |
| **E** | Emotion (감정) | Plutchik 8정서 기반 감정 코어 | 주감정 + 보조감정 + 강도 선택 |
| **R** | Crisis (위기) | 핵심 갈등과 동력 | [A] 내적 갈등 선택/서술, [B] WebSearch로 실무 방법론 조사 |

## 5-Layer 페르소나 구조 (v2)

생성된 페르소나는 6개 계층으로 구성된다.

| Layer | 이름 | 내용 | 핵심 규칙 |
|-------|------|------|-----------|
| **0** | Core Rules | 절대 행동 규칙 5~7개 | 형용사 금지, 동사형만 허용 |
| **1** | Identity | 역할 + 기질 + 배경 | G+H 데이터에서 도출 |
| **2** | Expression Style | 말투 패턴 + 5시나리오 대사 예시 | 실제 발화문 필수 |
| **3** | Decision & Judgment | 우선순위, 거절 방식, 도전 반응 | R+G 데이터에서 도출 |
| **4** | Interpersonal | 윗사람/동료/아랫사람 행동 패턴 | G+E 데이터에서 도출 |
| **5** | Boundaries | 싫어하는 것, 거부, 회피 | R에서 역추론 |
| — | Correction 기록 | 교정 이력 누적 (최대 30개) | 자동 병합, Layer 0 승격 |

### Layer 0 예시

```
❌ "강하다", "카리스마 있다", "논리적이다"
✅ "질문받으면 반문으로 돌린다"
✅ "결론부터 말하고, 근거는 물어볼 때만 제시한다"
✅ "모르는 건 '아직 판단 안 했다'로 표현한다"
```

### Layer 2 대사 예시

```markdown
> 1. 기초적인 질문을 받았을 때:
> "그건 공식 문서에 다 있어. 3번 섹션 읽어보고 안 되면 다시 와."

> 3. 동의하지 않는 제안을 들었을 때:
> "의도는 알겠는데, 근거가 뭐야? 데이터 없이 감으로 가자는 거면 난 반대."
```

## 진화 시스템

### 교정 (Correction)

페르소나 사용 중 잘못된 행동이 나오면 즉시 교정할 수 있다.

```
유저: "이건 아니야. 이 캐릭터는 사과하지 않아."

→ 교정 규칙 생성:
  - [2026-04-14] [장면: 실수 지적 시] 사과하지 않는다, 대신 원인 분석으로 넘어간다
```

- 최대 30개 누적, 초과 시 유사 규칙 자동 병합
- 5개 이상 누적 시 반복 패턴을 Layer 0에 승격 제안

### 병합 (Merge)

새 자료를 기존 페르소나에 증분 추가할 수 있다.

- 입력: 텍스트, URL (WebFetch), 파일 경로 (Read)
- 기존 내용 절대 덮어쓰기 금지 (append only)
- 충돌 감지 시 유저에게 선택권 부여
