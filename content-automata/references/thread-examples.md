# 쓰레드 콘텐츠 퓨샷 레퍼런스

아래는 최종 결과물인 쓰레드 글의 예시다. 이 톤, 구조, 길이를 참고하여 콘텐츠를 작성한다.

> **NOTE**: 이 파일은 샘플 포맷이다. 실제 퓨샷 예시가 확보되면 교체할 것.

---

## 예시 1: 속보형 쓰레드

**유형**: 속보 (Breaking)
**주제**: Claude 4 출시

```
🔥 Claude 4가 출시됐습니다.

Anthropic이 오늘 Claude 4를 공개했어요.
이번 버전에서 달라진 핵심 3가지를 정리합니다.

1️⃣ 100만 토큰 컨텍스트 윈도우
이전 대비 5배 늘어난 컨텍스트.
긴 코드베이스 전체를 한 번에 넣고 작업할 수 있습니다.

2️⃣ 에이전트 모드 강화
도구 사용, 파일 편집, 터미널 실행을
자율적으로 판단하고 수행합니다.

3️⃣ 멀티모달 출력
텍스트뿐 아니라 이미지, 차트도 직접 생성.
보고서 자동화에 바로 활용 가능합니다.

실무에서 가장 체감되는 변화는
"컨텍스트 안에서 전부 해결된다"는 점이에요.

더 자세한 분석은 다음 글에서 이어갑니다.
```

## 예시 2: 칼럼형 쓰레드

**유형**: 칼럼 (Column)
**주제**: AI 에이전트가 바꾸는 업무 자동화의 현실
**앵글**: "도구가 아니라 동료가 되는 순간"

```
AI 에이전트, 도구에서 동료로 넘어가는 중입니다.

최근 일주일간 흥미로운 움직임 3가지가 있었어요.

첫째, n8n이 AI 에이전트 노드를 정식 출시했습니다.
노코드로 "판단하고 행동하는" 워크플로우가 가능해졌어요.
기존 자동화가 "if-then"이었다면,
이제는 "상황 보고 알아서"입니다.

둘째, Anthropic이 Claude Code에 백그라운드 에이전트를 추가했습니다.
사람이 자는 동안 코드 리뷰, 테스트, 리팩토링을 돌립니다.
"새벽에 일 시켜놓고 아침에 확인"이 현실이 됐어요.

셋째, 국내에서도 카카오톡 AI 자동화 사례가 급증하고 있습니다.
고객 문의 → 분류 → 답변 → 에스컬레이션까지
사람 개입 없이 처리하는 팀이 늘고 있어요.

공통점이 보이시나요?

"사람이 지시하면 AI가 실행한다"에서
"AI가 판단하고 사람이 확인한다"로 바뀌고 있습니다.

이 전환이 불편한 사람도 있을 거예요.
하지만 먼저 적응하는 사람에게는 시간이라는 자원이 돌아옵니다.

여러분의 업무에서 "에이전트에게 맡길 수 있는 것"은 뭔가요?
```

## 작성 가이드라인

### 톤 & 보이스
- 전문적이되 친근한 말투 (해요체)
- 정보 전달이 우선, 감정 과잉 지양
- 독자에게 질문을 던져 참여 유도
- 이모지는 구조 표시용으로만 사용 (과용 금지)

### 구조
- 첫 줄: 핵심 메시지 (훅)
- 중간: 근거/사례 2-3개 (넘버링 또는 단락 분리)
- 마지막: 인사이트 요약 또는 독자 참여 유도 질문

### 길이
- 속보: 150-250자 내외
- 칼럼: 300-500자 내외

### 금지 사항
- 출처 불명확한 수치 사용
- 과도한 전문 용어 (독자는 AI에 관심 있는 일반인~실무자)
- 클릭베이트성 제목
- 동일 패턴 반복 (7개 글이 모두 같은 구조면 안 됨)
